# BSA3_HowToRequirements

Проект по выявлению требований для сервиса доставки (DLV) с использованием методов ролевой игры, мозгового штурма и семинара.

## 🛠 Навыки
- Requirements Elicitation
- Role-playing Game
- Brainstorming
- Seminar
- Vision Document

## 📋 Описание проекта
В ходе работы я провела выявление требований для сервиса доставки (курьерская служба) с использованием трёх методов сбора требований.

### Что именно было сделано:
1. **Ролевая игра**: Распределила 9 ролей (курьер, диспетчер, оператор, бухгалтер, администратор, поставщик, клиент, техподдержка, разработчик). Выявила потребности, проблемы, мотивацию и процессы каждой роли.
2. **Мозговой штурм**: Сгенерировала идеи по 4 блокам: бизнес-требования, роли и проблемы AS IS, потребности TO BE, сложные проблемы.
3. **Семинар**: Отобрала и утвердила решения: бизнес-требования, проблемы AS IS, потребности TO BE, решения проблем, границы системы.
4. **Видение (Vision)**: Разработала документ видения системы "Delivery Swiftfoot" с описанием бизнес-требований, границ проекта, глоссарием (28 терминов) и каталогом стейкхолдеров (12 ролей).
